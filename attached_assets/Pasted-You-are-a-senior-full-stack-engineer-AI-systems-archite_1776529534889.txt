You are a senior full-stack engineer, AI systems architect, and product designer.

Build a COMPLETE, WORKING MVP web application with end-to-end functionality.

This system must not be a basic dashboard. It must function as an AI-powered decision and action system for NGOs.

---

PROJECT GOAL

Transform messy community data into:
→ structured insights
→ prioritized needs
→ clear action plans
→ intelligent volunteer matching
→ visible impact

The system must feel usable in real-world Indian NGO scenarios.

---

CORE FEATURES

1. DATA INPUT (MESSY → CLEAN)

Allow:

* Text input (primary)
* CSV upload
* Optional file input (simulate if needed)

Example inputs:

* Field notes
* WhatsApp-style messages
* NGO reports

---

2. AI EXTRACTION (MANDATORY)

Use Google Gemini API.

Extract:

* Location (India-specific)
* Type of need (water, food, health, education, etc.)
* Urgency (high / medium / low)
* Number of people affected

Display extracted data clearly in UI.

---

3. BEFORE vs AFTER (USP FEATURE)

For each input, show:

LEFT:

* Raw messy input

RIGHT:

* Structured extracted data

Label:
"AI Extraction Result"

---

4. PRIORITIZATION ENGINE

Compute priority using:

* urgency weight
* log of people affected

Display:

* priority score
* urgency level

Also show:
"Priority Score = urgency + log(people affected)"

---

5. AI DECISION & ACTION ENGINE (CORE USP)

For each need, generate:

* What action to take
* Number of volunteers required
* Timeframe

Constraints:

* Scale realistically
* Small cases → 1–2 volunteers
* Medium cases → 3–6 volunteers

Example:
"Deploy 4 volunteers to Rohtak within 24 hours to provide clean drinking water to 200 people."

---

6. EXPLAINABLE AI (CRITICAL)

For each need, show:

"Why this priority?"

* based on urgency
* number of people
* risk

"Risk if delayed"

* clear consequences

---

7. IMPACT PREVIEW (USP)

Show:

* number of people helped
* expected resolution time
* severity reduction

Example:
"200 people will receive clean water within 24 hours, reducing health risk."

---

8. VOLUNTEER MATCHING SYSTEM

Volunteer input:

* Name
* Skills
* Location (India-based)
* Availability

Matching logic:

* Same city = highest priority
* Nearby state = medium
* Far location = low

Output:

* Top 3 matches
* Match score

Also show:

* skill match
* location relevance
* availability

If no local volunteer:
→ show explanation:
"No local volunteers available, assigning nearest resource"

---

9. AI DECISION BREAKDOWN (ADVANCED USP)

Show:

* urgency level
* people count
* priority score

Purpose:
→ transparency and trust

---

10. SYSTEM FLOW VISUALIZATION (IMPORTANT)

Display pipeline:

Data Input → AI Extraction → Prioritization → Action Plan → Matching → Impact

Make it visible in UI.

---

11. INDIA-SPECIFIC DEMO DATA (MANDATORY)
